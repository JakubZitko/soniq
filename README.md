# Soniq

Procedural UI sounds for the web. No audio files, no licensing headaches, no heavy asset packs — just clean, customizable sonic feedback generated by the Web Audio API.

[![CI](https://github.com/JakubZitko/soniq/actions/workflows/ci.yml/badge.svg)](https://github.com/JakubZitko/soniq/actions/workflows/ci.yml)

## Install

```bash
npm install @jakubzitko/soniq
```

Or use a CDN:

```html
<script src="https://unpkg.com/@jakubzitko/soniq/dist/soniq.iife.js"></script>
<script>
  Soniq.click();
</script>
```

## Why?

Most UI sound libraries ship WAV or MP3 packs. That means:

- Extra HTTP requests
- Licensing friction
- Limited customization
- Bloated bundles

Soniq generates every sound on demand. Each preset is a small function that schedules oscillators and noise buffers. The result is ~4KB of code and infinite variations.

## Usage

```js
import { click, hover, success, setVolume } from '@jakubzitko/soniq';

// Configure global volume
setVolume(0.4);

// Add feedback to a button
button.addEventListener('mouseenter', () => hover());
button.addEventListener('click', () => {
  click();
  success();
});
```

## Presets

| Preset | Description |
|--------|-------------|
| `click({ subtle })` | Standard or subtle button click |
| `hover({ subtle })` | Soft cursor-landing tone |
| `toggle(on)` | Switch toggle sound, direction-aware |
| `notification({ urgent })` | Alert chime |
| `success()` | Positive confirmation chord |
| `error()` | Failure buzz |
| `type({ subtle })` | Mechanical keystroke tick |
| `swipe({ direction })` | Gesture whoosh |
| `pop()` | Soft bubble pop |

## Advanced

Use the `Soniq` class to create isolated instances with their own volume and audio graph:

```js
import { Soniq } from '@jakubzitko/soniq';

const soniq = new Soniq({ volume: 0.3 });

soniq.click();
soniq.setVolume(0.6);
```

You can also build your own sounds from low-level primitives:

```js
soniq.tone({ freq: 880, duration: 0.2, type: 'sine', gain: 0.4 });
soniq.noise({ duration: 0.15, color: 'pink', gain: 0.2 });
```

## TypeScript

Soniq ships with TypeScript declarations. The package also exposes a typed `Soniq` class for custom instances.

## Browser support

Soniq uses the standard Web Audio API. It works in all modern browsers. Audio contexts are created lazily and resumed automatically on the first user interaction.
